# Contributing to linux-for-devops

This repo is built by students, for students. Contributions are welcome and encouraged.

---

## What You Can Contribute

- Fix a typo or incorrect command
- Add a missing concept or explanation
- Add a new lab task or project idea
- Add a resource link (book, video, course, tool)
- Add an interview question and answer
- Add a troubleshooting scenario
- Translate a note file (future goal)

---

## How to Contribute

1. Fork the repo
2. Create a branch: `git checkout -b fix/typo-in-permissions` or `feat/add-awk-examples`
3. Make your changes
4. Commit with a clear message: `fix: correct chmod octal example in note 03`
5. Push and open a Pull Request
6. Describe what you changed and why

---

## Guidelines

- Keep notes beginner-friendly — explain the why, not just the what
- Every command should have a comment explaining what it does
- Test commands before submitting — don't add untested examples
- One topic per PR — easier to review

---

## Questions?

Open an issue with the `question` label.
